# Samsam Product Identity

Samsam is not a health tracker.

Samsam is an emotional lifestyle world where healthy behaviour quietly becomes fuel for a personal living space.

## Product Sentence

Samsam helps users feel that their life is getting better by letting their companion, room, city memories, and lifestyle world visibly change through small healthy choices.

## What The User Returns For

The user should not return because they need to log calories.

They return because:

- their room might look different tonight
- their companion might react to them
- a food memory might appear
- the city might suggest a cozy place
- a small lifestyle choice might unlock atmosphere
- their world remembers them

## Product Pillars

1. Atmosphere before utility
2. Emotion before data
3. World before dashboard
4. Visible progression before abstract points
5. Food as memory, not only nutrition
6. Restaurants as lifestyle discovery, not only search
7. Companion as projection of self, not just a pet icon
8. Hidden gamification, not productivity pressure
9. Small interactions should change the environment
10. The app should feel like returning home

## Homepage Role

Homepage is the world entrance.

It should feel like:

- city night room
- lo-fi cozy game
- Persona-style menu mood
- Pinterest lifestyle board
- Animal Crossing room
- private emotional space

Homepage should not lead with:

- charts
- calorie rings
- macro dashboards
- task lists
- stacked feature cards

Health data may exist, but it should be emotionally translated into room light, companion energy, memories, and world changes.

## Navigation Philosophy

Primary navigation should reflect identity:

- World: living scene and companion
- City: restaurant/lifestyle discovery
- Memories: emotional scrapbook
- Self: settings, profile, body progress, saved places

Utility flows such as scan, meal log, and nutrition details should be reachable contextually, not treated as the product's main identity.

## Retention Loop

1. User opens the world
2. The atmosphere reflects time, mood, and recent behaviour
3. Companion reacts subtly
4. User leaves one small food/lifestyle memory
5. Room, companion, or city layer changes
6. Memory is saved
7. User has a reason to return tomorrow

## Dopamine Feedback Rules

Every meaningful action should trigger at least one environmental response:

- light warms
- companion moves/reacts
- room item appears
- postcard memory floats in
- background changes
- tiny collectible unlocks
- memory timeline updates

Avoid plain completion text as the only reward.

## Restaurant System Direction

Restaurants are not just listings.

They should become city discoveries:

- cozy places nearby
- mood-matched recommendations
- saved lifestyle postcards
- city-area unlocks
- meal memories
- aesthetic food moments
- companion reactions to places

## Companion Direction

The companion is not only a pet.

It is a projection of the user's lifestyle identity.

It should have:

- outfit changes
- body/stage changes
- room interactions
- routines
- mood
- speech restraint
- memory callbacks
- visible lifestyle progression

## Design Warning

If a screen starts to look like a dashboard, ask:

What changed in the user's world?

If the answer is "only a number," the experience is not Samsam enough.
